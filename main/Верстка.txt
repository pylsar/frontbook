Вёрстка - это создание веб страницы, по уже готовому макету.
Главные принципы Вёрстки:
	1)Совпадение с дизайном
	2)Кроссбраузерность(чтобы работал во всех браузерах)
	3)Поддержка популярных разрешений монитора(по вертикали особо нет разницы, а по горизонтали важно)
	4)Валидность(красота и правильность кода)

Термин «браузерный движок» получил распространение после того, как движки стали «отделимыми» от браузера. В число наиболее распространённых движков входят следующие:

Trident — проприетарный движок Microsoft Internet Explorer; используется многими программами для Microsoft Windows (например, мини-браузерами в программах Winamp и RealPlayer).
Gecko — открытый движок проекта Mozilla; используется в большом числе программ, основанных на коде Mozilla (браузере Firefox, почтовом клиенте Thunderbird, наборе программ SeaMonkey).
KHTML — разработан в рамках проекта KDE, используется в браузере Konqueror и послужил основой для WebKit.
WebKit — движок для браузера Apple Safari, включенного в операционную систему Mac OS X, и браузера Google Chrome (до 2013г). Встроен в библиотеку Qt.
Presto — проприетарный движок, разработанный Opera Software; являлся базой для браузера Opera до перехода на Blink, а также лицензирован для использования рядом сторонних компаний.
Blink — движок браузера Google Chrome с 28 версии, Opera c 15 версии и Vivaldi. Является ответвлением WebKit.
Edge — новый движок от компании Microsoft для её нового браузера Microsoft Edge. Является ответвлением Trident.

растровый рисунок: описывается пиксельной сеткой


для адаптивной вёрстки, надо для дива wrapper задавать max-width вместо width (это уберёт горизонтальную прокрутку)
для адаптивной вёрски блока с несколькими картинками:
	width и margin лучше писать в %(например margin 2% 4 блока, тогда width блока картнки будет 100-(2%margina *6)=88%)
	6 так как margin у каждого блока в обе стороны, но у крайнего слева и справа нет(они упираются в родительский блок)
	блоку с картинками надо дать  box-sizing: border-box(чтобы падинги и марджины не влияли на размер блока)
	родительскому блоку(который объединяет все картинки) надо дать отрицательные марджины (margin: 0 -2%;), чтобы крайний левый и крайний правый марджин не отталкивались,
	а проваливались в родительский блок.

смысл адаптивки: для каждого размера прописываем новый медиазапрос, где меняем поведения блоков
для адаптивной вёрстки: height нежелательно прописывать(исключение логотип), если прям так нужно прописать высоту, то пишем min-height
background-color: rgba(#fff , 0.2); хек код можно использовать в rgba

Вёрстку лучше всего начинать с типографики: определить главный шрифт, размер шрифта, высоту строки.
